Difference between TCP and UDP?

TCP is connection-oriented protocol that ensure reliable data delivery. slower delivery
used in:- SSH → Port 22
	HTTP → Port 80
	HTTPS → Port 443
	FTP → Port 21
	MySQL → Port 3306

UDP is connectionless and faster but does not guarantee data delivery.
used in:- DNS → Port 53
	Video streaming
	Online gaming
	VoIP calls

AWK - Cut the file column and row wise

Link - To create a shortcut for a file or directory

ln <original-file-path> <hardlink-path> --> To create a hard link
ln -s <original-file-path> <softlink-path> --> To create a soft/symbolic link

Example:

ln -s d1/d2/d3/d4/original softlink

A softlink will point to the path of the original file. As soon as the original file get removed or moved - The softlink stops working

Whereas the hardlink directly points to the memory location of the original file. Even if the original file gets removed or moved, the hardlink will still work

HardLink Only works for files, Whereas softlink can be used for directories as well

kill <PID> --> To terminate a process [Signal 15]
Terminating a process means stopping a running program gracefully.

The process gets a signal and has a chance to:
*save data
*close files properly
*exit safely

kill -9 <PID> --> To Forcefully kill a process
Forcefully killing a process means immediately stopping a process without giving it time to save or clean up.

SSH -- Secure SHell or Secure Socket Shell

SSH is a network protocol that enables an user to access a remote server in a secure way

Default Port - 22

Jenkins -- 8080
Tomcat -- 8080
HTTP -- 80
HTTPS -- 443
Nginx -- 80
Kubernetes API Server -- 6443

Port: A port is a virtual point where network connections start and end

SSH Commands:

--------------------------------------------------------------------------------------------------------------
SSH Passwordless Connection

To establish a passwordless connection, Generate a key pair using ssh-keygen command. 
Copy the contents of the newly generated public key to the authorized keys file of the remote server under .ssh folder.

ssh <username>@<ip-address> --> To SSH without provided key file 

Example:

ssh ubuntu@172.31.36.45
